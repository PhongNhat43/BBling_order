## 📋 **CÁC LỖI ĐÃ ĐƯỢC SỬA**

### ✅ **Bug #1: Khách gửi tin nhưng không thấy hiển thị**
**Triệu chứng**: Khách nhập ký tự, bấm Gửi nhưng tin nhắn không hiển thị trong chat log

**Nguyên nhân**: 
- Hàm `sendText()` trong chat.js chỉ gửi message lên Firebase
- KHÔNG render message locally cho customer xem
- Customer phải đợi snapshot listener bắn lại mới thấy tin (nhưng listener đã filter bỏ customer messages)

**Giải pháp**:
```javascript
// TRƯỚC (SAI):
if (this.useFirebase) {
  db.collection(...).add({...}); // Chỉ gửi, không render
  return; // Return luôn!
}

// SAU (ĐÚNG):
const node = el('div', null, text);
const { status } = this.pushUserBubble(node, 'Đang gửi...'); // Render local TRƯỚC

if (this.useFirebase) {
  db.collection(...).add({...}).then(() => {
    status.textContent = 'Đã gửi'; // Update status
  });
}
```

### ✅ **Bug #2: Admin thấy tin nhắn bị lặp**
**Triệu chứng**: Khách gửi "Tuấn Nhật" → Admin thấy "Tuấn Nhật" rồi lại thấy "nhật" lặp lại bên dưới

**Nguyên nhân**:
- Admin's `renderDetail()` dùng `snap.docs.forEach()` thay vì `snap.docChanges()`
- Mỗi lần snapshot fire → Clear toàn bộ chat → Re-render TẤT CẢ messages
- Gây hiệu ứng flicker/duplication khi messages arrive nhanh

**Giải pháp**:
```javascript
// TRƯỚC (SAI):
onSnapshot(function (snap) {
  chat.innerHTML = ''; // Clear all
  snap.docs.forEach(function (doc) { // Re-render ALL
    // ...render message...
  });
});

// SAU (ĐÚNG):
onSnapshot(function (snap) {
  snap.docChanges().forEach(function (change) { // Only process changes
    if (change.type === 'added') {
      // Append new message
      chat.appendChild(wrapper);
    } else if (change.type === 'removed') {
      // Remove deleted message
      msgEl.remove();
    }
  });
});
```

**Cải tiến thêm**:
- Thêm `data-msg-id` để track từng message
- Hỗ trợ xóa tin nhắn riêng lẻ (type === 'removed')
- Auto-scroll mượt mà hơn

### ✅ **Bug #3: Customer chỉ thấy reply của admin, không thấy toàn bộ chat**
**Triệu chứng**: Sau khi admin reply → Customer thấy reply nhưng không thấy lịch sử chat đầy đủ

**Nguyên nhân**:
- Listener ở customer side filter quá chặt:
  - `if (d.from === 'admin')` → Chỉ render admin messages
  - Customer messages đã render local nhưng bị filter trong snapshot
  - Khi reload page → Mất hết customer messages cũ

**Giải pháp**:
- Customer messages: Render local ngay khi gửi (không đợi snapshot)
- Admin messages: Render từ snapshot listener
- Kết hợp 2 cách này → Customer thấy đầy đủ cuộc trò chuyện

**Lưu ý**: 
- `lastProcessedTime` filter vẫn giữ để tránh hiển thị messages cũ từ session trước
- Chỉ render admin messages từ snapshot để tránh duplicate
